Contratar o VPS na Hostinger

🛒 Comprar o servidor

Acesse o site da Hostinger

Va em hostinger.com.br/servidor-vps e escolha o plano KVM 1 (mais barato, suficiente para comecar).

Finalize o pagamento

Escolha o periodo de cobranca (quanto maior, mais desconto) e finalize.

Escolha o Sistema Operacional

No painel do VPS, selecione Ubuntu 22.04.

Defina a senha de root

Crie uma senha forte. Exemplo: MinhaSenh@VPS2026

Anote o IP do servidor

Apos a instalacao, o painel mostra o IP. Exemplo: 185.200.100.50. Anote!

⚠
Anote em local seguro: o IP do VPS e a senha de root. Voce vai precisar deles em todas as etapas.
2

Registrar Dominio e Apontar DNS

🌐 Comprar o dominio

Registre seu dominio (ex: gestescolar.com.br) na Hostinger ou no registro.br.

🔌 Apontar DNS para o VPS

Isso diz ao mundo: "quando alguem digitar meu dominio, va para o IP do meu VPS".

No painel de DNS do seu dominio, adicione estes registros:

TipoNome (Host)ValorTTL
A@SEU_IP_DO_VPS3600
AwwwSEU_IP_DO_VPS3600
⏳
A propagacao do DNS pode levar de 15 minutos a 24 horas. Seja paciente.

✅ Verificar se o DNS propagou

No seu computador, abra o Terminal (ou Prompt) e digite:

nslookup gestescolar.com.br

Se mostrar o IP do seu VPS, esta pronto! Ou verifique em dnschecker.org.

3

Acessar o VPS via SSH

💻 O que e SSH?

SSH e como um "controle remoto" para o servidor. Voce digita comandos no seu computador, e eles sao executados no VPS.

🔌 Conectar ao VPS

No Windows 10/11, abra o Terminal (ou PowerShell) e digite:

ssh root@SEU_IP_DO_VPS

Primeira vez: confirmar conexao

Vai perguntar "Are you sure...?". Digite yes e aperte Enter.

Digitar a senha

Digite a senha de root (os caracteres nao aparecem na tela, e normal). Aperte Enter.

Pronto!

Voce vera algo como root@vps-123456:~#. Agora voce controla o VPS!

💡
Alternativa visual: No painel da Hostinger, clique em VPS > Terminal/Console para acessar sem instalar nada.
4

Instalar Tudo no VPS

Execute os comandos abaixo um por um dentro do VPS (via SSH):

📦 Atualizar o sistema

apt update && apt upgrade -y

Pode demorar 1-2 min. Se perguntar algo, aperte Enter.

🌎 Instalar Nginx (servidor web)

apt install -y nginx

📁 Instalar Git

apt install -y git

🔒 Instalar Certbot (HTTPS gratuito)

apt install -y certbot python3-certbot-nginx

🛡 Configurar Firewall

ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

✅ Verificar se Nginx esta rodando

systemctl status nginx

Deve mostrar active (running) em verde.

5

Enviar os Arquivos do GestEscolar

📁 Criar a pasta do site no VPS

No VPS (via SSH):

mkdir -p /var/www/gestescolar

📦 Escolha como enviar os arquivos

Se o projeto esta no GitHub, execute no VPS:

cd /var/www
git clone https://github.com/SEU_USUARIO/gestescolar.git
👍
Para atualizar no futuro, basta: cd /var/www/gestescolar && git pull

No seu computador (NAO no VPS), abra um terminal e execute:

scp -r index.html css js assets root@SEU_IP:/var/www/gestescolar/
⚠
NAO envie as pastas deploy/, supabase/, .claude/ ou .git/.

Baixe o WinSCP

Acesse winscp.net e instale o programa (gratuito).

Conecte ao VPS

Protocolo: SFTP | Host: IP do VPS | Usuario: root | Senha: sua senha.

Navegue ate a pasta

No lado direito (VPS), va ate /var/www/gestescolar/.

Arraste os arquivos

Do lado esquerdo (seu PC), arraste: index.html, css/, js/, assets/.

⚠
NAO envie: deploy/, supabase/, .claude/, .git/, bundle.html.

✅ Verificar os arquivos no VPS

ls -la /var/www/gestescolar/

Deve mostrar: index.html, css/, js/, assets/.

6

Configurar o Nginx

📝 Criar arquivo de configuracao

No VPS, execute:

nano /etc/nginx/sites-available/gestescolar

Isso abre um editor de texto. Cole o conteudo abaixo (substitua o dominio pelo seu):

server {
    listen 80;
    server_name gestescolar.com.br www.gestescolar.com.br;

    root /var/www/gestescolar;
    index index.html;

    # Seguranca
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    # SPA: qualquer rota cai no index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache para arquivos estaticos
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # Bloquear pastas sensiveis
    location ~* /(supabase|deploy|\.claude|\.git) {
        deny all;
        return 404;
    }

    # Compressao
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
    gzip_min_length 256;
}
📝
Como salvar no nano:
1. Aperte Ctrl + O (salvar) > Enter
2. Aperte Ctrl + X (sair)

🔌 Ativar o site e reiniciar

# Ativar o site
ln -sf /etc/nginx/sites-available/gestescolar /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# Testar configuracao
nginx -t

# Reiniciar Nginx
systemctl restart nginx

O nginx -t deve mostrar: "syntax is ok" e "test is successful".

✅ Testar no navegador

Abra o navegador e acesse:

http://SEU_IP_DO_VPS

Deve aparecer a tela de login do GestEscolar! Se o DNS ja propagou:

http://gestescolar.com.br

Ativar HTTPS / SSL (Cadeado Verde)

⚠
IMPORTANTE: So faca isso APOS o DNS estar propagado (o dominio ja abre no navegador via HTTP).

No VPS, execute:

certbot --nginx -d gestescolar.com.br -d www.gestescolar.com.br

Vai fazer algumas perguntas:

PerguntaResponda
Enter email addressSeu email (para avisos de renovacao)
Agree to terms?Y
Share email?N
Redirect HTTP to HTTPS?2 (redirecionar)
🔒
Pronto! Agora https://gestescolar.com.br funciona com cadeado verde. O certificado renova automaticamente a cada 90 dias.
8

Configurar Dominio no Supabase

Apos o site estar no ar, atualize o Supabase para reconhecer seu dominio:

Acessar o painel Supabase

Va em supabase.com/dashboard e selecione seu projeto.

Ir em Authentication > URL Configuration

No menu lateral, clique em Authentication, depois URL Configuration.

Atualizar as URLs

Preencha:

CampoValor
Site URLhttps://gestescolar.com.br
Redirect URLshttps://gestescolar.com.br/**

Salvar

Clique em Save.

Como Atualizar o Site no Futuro

Sempre que fizer mudancas no codigo e quiser publicar:

Conecte no VPS e execute:

ssh root@SEU_IP
cd /var/www/gestescolar
git pull

Pronto! As mudancas ja estao no ar.

No seu computador, execute:

scp -r index.html css js assets root@SEU_IP:/var/www/gestescolar/

Abra o WinSCP, conecte, e arraste os arquivos atualizados para /var/www/gestescolar/.

Problemas Comuns e Solucoes

❌ "Connection refused" ao acessar pelo navegador

# Verificar se Nginx esta rodando:
systemctl status nginx

# Verificar firewall:
ufw status

❌ "502 Bad Gateway"

# Verificar se os arquivos existem:
ls /var/www/gestescolar/

# Testar configuracao do Nginx:
nginx -t

❌ Site carrega mas Supabase nao conecta

Abra o console do navegador (F12 > Console) e veja o erro. Certifique-se que o dominio esta liberado no Supabase (Parte 8).

❌ "DNS_PROBE_FINISHED_NXDOMAIN"

O DNS ainda nao propagou. Aguarde ate 24h. Verifique em dnschecker.org.

🔑 Esqueci a senha do VPS

No painel Hostinger > VPS > Reset Root Password.

🔄 Como reiniciar o Nginx / ver logs

# Reiniciar Nginx:
systemctl restart nginx

# Ver logs de erro:
tail -50 /var/log/nginx/error.log

# Ver logs de acesso:
tail -50 /var/log/nginx/access.log
